도구랩 검색엔진 노출 설정 가이드

기본 반영 완료
- robots.txt 생성
- sitemap.xml 생성
- 각 페이지 canonical 추가
- Open Graph / Twitter 카드 메타 추가
- 기본 favicon / social image / manifest 추가
- 구조화 데이터(JSON-LD) 추가

배포 후 해야 할 일
1. 사이트를 GitHub Pages에 올립니다.
2. Google Search Console에서 속성을 추가합니다.
3. sitemap.xml 을 제출합니다.
   - 제출 주소: https://mizunomeru537.github.io/MizuNomeru5379/sitemap.xml
4. 홈(/)과 핵심 도구 몇 개를 URL 검사로 색인 요청합니다.
5. Bing Webmaster Tools에도 같은 sitemap.xml 을 제출합니다.

중요한 점
- 현재 사이트는 GitHub Pages의 하위 경로(https://mizunomeru537.github.io/MizuNomeru5379/)에 있습니다.
- Google의 사이트명(site name)과 검색결과 favicon은 도메인/서브도메인 단위 기준이라 하위 경로 사이트에서는 제한될 수 있습니다.
- 검색결과에서 브랜드 노출을 더 안정적으로 가져가려면 custom domain 연결이 가장 좋습니다.

추가로 추천
- 새 도구를 만들 때마다 sitemap.xml 도 같이 갱신
- 각 도구마다 300~700자 설명 블로그 페이지 추가
- 한 URL에서 언어를 JS 탭으로만 바꾸는 구조보다, 나중에는 언어별 별도 URL(/en/, /ja/ 등)로 분리하면 SEO에 더 유리
