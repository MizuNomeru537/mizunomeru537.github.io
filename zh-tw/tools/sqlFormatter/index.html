Long SQL queries are reorganized with a consistent readable layout, cleaner SELECT lists, readable JOIN blocks, and safer keyword handling for identifiers like REGION_NAME or COUPON_ORDER_COUNT.

What changed

  • Indentation is fixed to 4 spaces across the formatter.
  • Keyword detection is token-based, so ON / OR / AND inside identifiers are less likely to break names such as REGION_NAME.
  • Very vendor-specific syntax can still need manual cleanup, but long everyday queries should read much better now.

How to use

  1. Paste your SQL query into the source box.
  2. Choose keyword case and click Format.
  3. Copy the result for debugging, documentation, or tickets.

FAQ

Does it upload my SQL?

No. Formatting runs locally in your browser for quick day-to-day work.

Why is the layout fixed?

This page now uses one consistent style so long SQL stays easy to scan instead of changing layout from query to query.

This formatter is designed for practical readability first. If a vendor-specific query still needs a tiny manual touch-up afterward, that is normal.
